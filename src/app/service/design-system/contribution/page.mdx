# 기여 가이드

디자인 시스템은 협업으로 성장합니다. 아래 원칙을 따릅니다.

## 워크플로우

1. 이슈 등록: 문제/제안 배경과 사용 맥락 설명
2. 디자인 제안: 원칙/토큰 영향도/대안 비교 포함
3. 구현/리뷰: 컴포넌트 API·문서·데모 포함 PR
4. 배포/공지: 변경 로그와 마이그레이션 가이드 제공

## 품질 기준

- 접근성: 명도 대비, 포커스 표시, 키보드 네비게이션
- 문서화: 예제·코드 샘플·스크린샷/애니메이션
- 테스트: 스냅샷·상태/상호작용 단위 테스트
