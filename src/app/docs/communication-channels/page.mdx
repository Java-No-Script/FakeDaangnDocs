# 커뮤니케이션 채널 가이드

<Callout emoji="🗣️">
  빠르고 투명한 협업을 위해 채널을 목적별로 운영합니다. 필요한 채널은 즐겨찾기에
  고정하고 알림 구독을 조정하세요.
</Callout>

## 공지 채널 (읽기 전용)

- `#공지-전체`: 전사 필수 공지
- `#인사-공지`: 제도/휴가/경조사/복지 업데이트
- `#보안-공지`: 보안 점검/사고 대응/권고 사항

## 문의/지원 채널

- `#IT운영지원팀`: 장비/계정/네트워크/회의실 화면/소프트웨어 설치
- `#보안-문의`: 권한/접근/보안교육/사고 신고
- `#재무-정산-문의`: 영수증/법인카드/지출결의/세금계산서
- `#문의-휴가`: 휴가 시스템/신청/정책 해석
- `#문의-경조사`: 경조금/증빙/일수 확인

## 업무 협업 채널 예시

- 제품/개발: `#product`, `#backend`, `#frontend`, `#design-system`
- 데이터/분석: `#data-platform`, `#experiment`, `#ml-platform`
- 서비스 운영: `#cs-ops`, `#safety-ops`, `#community-ops`

## 채널 운영 원칙

- 제목에 목적/요약을 포함하고, 본문에는 배경/현상/제안/결정 요청을 구분합니다.
- 결정은 쓰레드 첫 댓글에 최종안으로 명시하고 문서 링크를 남깁니다.
- 이슈 상태(열림/대기/해결)를 메시지 리액션으로 표시합니다.
- 민감 정보는 전용 툴과 권한 체계를 사용하고, 채널에는 요약만 남깁니다.

## 알림/즐겨찾기 팁

- 팀/공지/문의 채널은 즐겨찾기 고정, 잔여 채널은 키워드 알림으로 전환합니다.
- 방해금지 시간(포커스 타임)에는 채널 알림을 일괄 일시중지합니다.
