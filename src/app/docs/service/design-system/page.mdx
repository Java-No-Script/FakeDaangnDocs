## 버전 · 릴리스

<Steps>
### 버전 정책(SemVer)

fix = patch, non‑breaking feature = minor, breaking = major

### 변경 로그

모든 변경은 문서/마이그레이션 가이드를 포함합니다.

### 릴리스 주기

주 1회 정기 릴리스, 긴급 수정은 hotfix로 별도 배포합니다.

</Steps>
